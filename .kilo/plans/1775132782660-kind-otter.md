# Mentor Dashboard - Complete Analysis & Production-Level Improvement Plan

## 1. CURRENT FLOW OVERVIEW

```
Mentor Dashboard (/mentor-dashboard/)
├── Overview       → Stats, Analytics, Sessions, Requests, Quick Actions
├── My Profile     → Editable profile form + preview card
├── Courses        → Course CRUD + Lesson builder
├── Messages       → Redirects to /messages (shared)
├── Schedule       → Weekly calendar availability manager
├── Reviews        → Rating overview + paginated review list
└── Notifications  → Real-time notification feed with accept/reject
```

**Guard Chain:**
- Frontend: `MentorDashboardLayout.jsx` checks `user.mentorStatus === 'approved'`
- Backend: `protect` (JWT) → `requireMentor` (mentorStatus check)

## 2. TAB-BY-TAB ANALYSIS

### 2.1 Overview Tab (`MentorOverview.jsx`)
**Data Sources (real API calls):**
- `GET /mentor/dashboard/stats` → Total Students, Active Sessions, Rating, Earnings
- `GET /mentor/dashboard/upcoming-sessions` → Booked schedule slots
- `GET /mentorship/requests/incoming` → Pending mentorship requests
- `GET /mentor/dashboard/analytics` → Student growth, completion rate, per-course stats

**Strengths:**
- Real data from MongoDB aggregations (no fake data)
- Stat cards are clean and informative
- Analytics charts (student growth bar chart, completion donut, per-course table)
- Quick actions for common tasks
- Upcoming sessions and pending requests panels

**Weaknesses:**
- No date range selector for analytics (hardcoded 30d default)
- No earnings chart (data is fetched but not displayed in a chart)
- No comparison with previous period (growth % indicators)
- No course-specific drill-down from analytics table
- Pending requests don't show accept/reject actions inline (only in Notifications tab)
- No "this week" summary or activity heatmap
- The `SimpleBarChart` is basic - no tooltips, no proper axis labels

---

### 2.2 My Profile Tab (`MentorProfile.jsx`)
**Data Sources:**
- `GET /mentor/profile` → Full mentor profile
- `PUT /mentor/profile` → Save updates

**Strengths:**
- Profile completion progress bar
- Live preview card showing how students see the mentor
- Skills with level (Beginner/Intermediate/Expert)
- Portfolio links (GitHub, LinkedIn, website)

### 2.3 Courses Tab (`MentorCourses.jsx`)

**Strengths:**

- Lesson builder with title, duration, video URL, description
- Draft/Published status management
- Clean card-based course list with stats (lessons, students, rating)

**Weaknesses:**
- **CRITICAL: No price input field in the form** - the `price` field exists in the model but the UI doesn't expose it
- **No thumbnail upload** - only a text URL field
- **No drag-and-drop lesson reordering**
- **No lesson editing after creation** - must delete and re-add
- **No bulk actions** (publish all, delete multiple)
- **No course duplication/copy feature**
- **No draft auto-save** - if you navigate away, unsaved changes are lost
- **No course preview** - can't see how the course looks to students
- **No enrollment analytics per course** from this page
- **No course review management** (reply to reviews)
- Lessons are added sequentially (slow for many lessons)
- No rich text editor for lesson content
- No course prerequisites/dependencies
- No course visibility settings (public/unlisted/private)
- No course scheduling (start date, enrollment deadline)

## 3. COURSE CREATION FLOW (Free vs Paid)

### Current Flow:
```
1. Mentor clicks "Create Course"
2. Form opens: Title, Description, Category, Level, Tags, Thumbnail (URL only)
3. Add lessons (title, duration, video URL, description)
4. Click "Create Course" → POST /api/courses
5. Lessons added sequentially → POST /api/courses/:id/lessons
6. Course created with status='draft', price=0
7. Mentor clicks "Publish" → PUT /api/courses/:id/publish
```

### Critical Issues:
1. **NO PRICE INPUT** - The form doesn't have a price field. Paid courses cannot be created through the UI.
2. **NO PAYMENT INTEGRATION** - Even if price is set via API, enrollment has no payment check. Anyone can enroll in paid courses for free.
3. **NO COURSE CATEGORIES MANAGEMENT** - Categories are hardcoded in frontend.
4. **NO THUMBNAIL UPLOAD** - Only URL input, no file upload.
5. **NO COURSE PREVIEW** - Mentor can't see how the course looks to students.
6. **NO AUTO-SAVE** - Draft changes are lost if user navigates away.
